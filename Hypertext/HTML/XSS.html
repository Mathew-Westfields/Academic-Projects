Definition

Cross-Site Scripting (XSS) ist eine Form Injektions Attacke. Alle Injektions Attacken versuchen, per User-Input dem Server/Compiler Instruktionen unter zu schieben ,die er eigentlich nicht auswerten sollte. Die zwei populärsten Attacken im Netz sind beide Injektions Attacken. Cross-Site Scripting im Allgemeinen versuchen alle irgendwie ein <script> some_JavaScript_function(); </script> zu injizierten. Das heißt es ist eine HTML-Injektion die es wiederrum erlaubt Javascript auszuführen. Dies kann auf eine Menge Wege passieren, diese Form der Attacke nutzt sämtlichen nicht "sanitären"-Userinput aus. Und wie man gleich und spätestens nach der Geschichte von JS.Spacehero merkt kann JavaScript ziemlich gefährliche Sachen machen.

JavaScript

das Problem mit JS

JavaScript ist ein Programmiersprache die vollständig im Browser implementiert läuft. Dieser Fakt macht es in der Informationszeit wo fast alles Internet Anbindung hat zu einem verstörend mächtigem Werkzeug. Und diese Formulierung ist nicht hyperbolisch, JavaScript diktiert was der Browser tut und obendrein ist es Überall. Jedes Mal wenn ein Webpage im Hintergrund sich leise refreshed oder irgendwas passiert ohne dass die Seite komplett neu geladen werden muss hat JavaScript seine Finger im Spiel. Und das aller schlimmste daran ist ,dass JavaScript leise im Hintergrund ausgeführt wird. Das kann man auch nicht "reparieren" das ist eine definierende Eigenschaft weshalb es benutzt wird.

Ein Beispiel

Man stelle sich vor ich habe mein php verbockt.(Php benutzt man für viele Dinge unter anderem um User-Input auszuwerten) Ich habe meine eigene kleine Webseite geschrieben, die einem einfach nur Hallo sagt. Man schreibt seinen Vornamen in ein Eingabefeld, schickt es ab und das Programm sagt einem Hallo "Vorname". Das sind umgerechnet ca. 3-4 Zeilen php nicht viel Platz für Fehler würde man denken. Nur blöd ist, dass mein hypothetischer Php Code den User-Input roh entgegen nimmt. Ziemlich schlecht gelaufen, ein neugieriger Mensch gibt <script> alert(""); </script> ein und das alert fenster poppt auf. Der neugierige Mensch hat auch noch ne kriminelle Ader und hinterlässt nen JavaScript, dass jedem Browser sagt: "bitte einmal alle Session-Cookies und Passwörter die du hast! Danke." Das heißt ,jeder Benutzer der auf diese Seite klickt ist jetzt seine Session-Cookies , gespeicherten Passwörter usw. los. Und das alles ohne das der Browser auch nun piep macht.
